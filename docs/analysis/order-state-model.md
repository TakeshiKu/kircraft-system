
# Order State Model — Kircraft System

## Назначение

Документ фиксирует модель состояний заказа и допустимые переходы между статусами.

---

## Статусы заказа

- Создан
- Подтвержден
- Требует уточнения
- Оплата получена
- В работе
- Отправлен
- Отменен

---

## Правила переходов

- Новый заказ создается в статусе «Создан»
- Заказ в статусе «Создан» может быть подтвержден, отменен или переведен в статус «Требует уточнения»
- Заказ в статусе «Подтвержден» может быть оплачен или отменен
- Заказ в статусе «Требует уточнения» после получения уточнений возвращается на повторное рассмотрение
- Заказ в статусе «Оплата получена» может быть переведен в статус «В работе»
- Заказ в статусе «В работе» может быть переведен в статус «Отправлен»

---

## Диаграмма состояний заказа

```mermaid
stateDiagram-v2
    [*] --> Создан

    Создан --> Подтвержден : заказ подтвержден
    Создан --> Требует_уточнения : требуется уточнение
    Создан --> Отменен : заказ отменен

    Требует_уточнения --> Подтвержден : уточнение получено

    Подтвержден --> Оплата_получена : заказ оплачен
    Подтвержден --> Отменен : заказ отменен

    Оплата_получена --> В_работе : заказ взят в работу
    В_работе --> Отправлен : заказ отправлен
