# Order State Model — Kircraft System

## Назначение

Документ фиксирует модель состояний заказа и допустимые переходы между статусами в системе Kircraft.  
Модель используется для описания жизненного цикла заказа и служит основанием для реализации бизнес-логики системы.

---

# Статусы заказа

| Статус | Описание |
|------|------|
| Создан | Заказ оформлен клиентом и ожидает рассмотрения представителем мастерской |
| Подтвержден | Заказ проверен и подтвержден представителем мастерской |
| Требует уточнения | Для выполнения заказа требуется дополнительная информация от клиента |
| Оплата получена | Система зафиксировала успешную оплату заказа |
| В работе | Заказ принят в выполнение |
| Отправлен | Заказ выполнен и отправлен клиенту |
| Отменен | Заказ отменен клиентом или представителем мастерской |

---

# Таблица переходов состояний

| Текущее состояние | Следующее состояние | Событие |
|------|------|------|
| Создан | Подтвержден | заказ подтвержден |
| Создан | Требует уточнения | запрошено уточнение |
| Создан | Отменен | заказ отменен |
| Требует уточнения | Подтвержден | уточнение получено |
| Подтвержден | Оплата получена | заказ оплачен |
| Подтвержден | Отменен | заказ отменен |
| Оплата получена | В работе | заказ взят в работу |
| В работе | Отправлен | заказ отправлен |

---

# Ограничения переходов

1. Новый заказ создается в статусе **Создан**.  
2. Переходы между состояниями возможны только в соответствии с таблицей переходов.  
3. После перехода заказа в статус **Оплата получена** состав заказа не может быть изменен.  
4. Заказ не может быть отменен после успешной оплаты.  
5. Заказ может быть переведен в статус **В работе** только после получения оплаты.

# Диаграмма состояний заказа

```mermaid
stateDiagram-v2
    [*] --> Created

    state "Создан" as Created
    state "Подтвержден" as Confirmed
    state "Требует уточнения" as NeedsClarification
    state "Оплата получена" as Paid
    state "В работе" as InProgress
    state "Отправлен" as Shipped
    state "Отменен" as Cancelled

    Created --> Confirmed : подтверждение
    Created --> NeedsClarification : запрос уточнения
    Created --> Cancelled : отмена

    NeedsClarification --> Confirmed : уточнение получено

    Confirmed --> Paid : оплата
    Confirmed --> Cancelled : отмена

    Paid --> InProgress : взят в работу
    InProgress --> Shipped : отправка
