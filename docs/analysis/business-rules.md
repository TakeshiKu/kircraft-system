
# Business Rules — Kircraft System

## Перечень бизнес-правил

| Код | Правило | Условие | Связанные Use Cases |
|----|----|----|----|
| BR-01 | Оформление заказа возможно только при наличии товаров в корзине | Система не позволяет перейти к оформлению заказа, если корзина пуста | UC-07, UC-08 |
| BR-02 | Заказ создается только после подтверждения оформления | Если клиент не подтверждает оформление заказа, заказ не создается | UC-08 |
| BR-03 | Номер телефона является обязательным | Клиент не может завершить оформление заказа без указания номера телефона | UC-08 |
| BR-04 | Данные доставки являются обязательными | Клиент должен указать город и адрес доставки перед подтверждением заказа | UC-08 |
| BR-05 | Каждому заказу присваивается уникальный идентификатор | При создании заказа система формирует уникальный номер заказа | UC-08, UC-16 |
| BR-06 | После оформления заказ получает статус «Создан» | Новый заказ фиксируется системой со статусом «Создан» | UC-08 |
| BR-07 | Заказ должен быть рассмотрен представителем мастерской | Заказ может быть подтвержден, отклонен или отправлен на уточнение | UC-09, UC-A1 |
| BR-08 | Оплата доступна только после подтверждения заказа | Клиент может перейти к оплате только после перевода заказа в статус «Подтвержден» | UC-09, UC-15 |
| BR-09 | Заказ можно отменить только до оплаты | Если заказ еще не оплачен, клиент может отменить его | UC-17 |
| BR-10 | Изменение статуса заказа возможно только по допустимым переходам | Система должна ограничивать переходы между состояниями заказа в соответствии с моделью состояний | UC-09, UC-10, UC-11, UC-A2, UC-A3, UC-A4 |
| BR-11 | Заказ может перейти в статус «Оплата получена» только после успешной оплаты | Система фиксирует оплату только после подтверждения платежа | UC-15, UC-11, UC-A2 |
| BR-12 | Заказ может перейти в статус «В работе» только после оплаты | Представитель мастерской может взять заказ в работу только после оплаты | UC-A3 |
| BR-13 | Заказ может быть отправлен только после начала выполнения | Отправка заказа возможна только для заказов в статусе «В работе» | UC-A4 |
| BR-14 | Клиент получает уведомления при изменении статуса заказа | Система отправляет уведомления при подтверждении, оплате, начале работы, отправке и запросе уточнения | UC-09, UC-10, UC-11 |
| BR-15 | Клиент может просматривать только свои заказы | В разделе «Мои заказы» отображаются только заказы текущего пользователя | UC-16 |
| BR-16 | Изменение корзины пересчитывает итоговую стоимость | При изменении количества товаров или состава корзины система пересчитывает стоимость корзины | UC-07 |
| BR-17 | Уточнение заказа переводит его на повторное рассмотрение | После получения уточнений заказ возвращается на этап рассмотрения | UC-10, UC-A1 |
| BR-18 | После оплаты состав заказа не может быть изменен | Система не позволяет изменять состав оплаченного заказа | UC-15, UC-16, UC-17, UC-A3 |
