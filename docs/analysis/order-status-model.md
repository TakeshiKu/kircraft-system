# Order Status Model

## Назначение

Документ определяет перечень статусов заказа в системе Kircraft и правила перехода между ними.  
Статусы используются для отображения состояния заказа клиенту и для управления процессом обработки заказа внутри системы.

---

# Перечень статусов заказа

| Статус | Описание |
|------|------|
| Создан | Заказ создан клиентом. Содержимое заказа сформировано, но еще не подтверждено мастером. |
| Требует уточнения | Для выполнения заказа требуется дополнительная информация от клиента (например цвет, размер или персонализация изделия). |
| Подтвержден | Мастер подтвердил возможность выполнения заказа и согласовал параметры заказа. |
| Оплата получена | Оплата по заказу успешно получена и подтверждена платежной системой. |
| В работе | Заказ принят в производство и находится в процессе изготовления. |
| Отправлен | Заказ передан в службу доставки и отправлен клиенту. |
| Отменен | Заказ отменен клиентом или мастером и не будет выполняться. |

---

# Основной сценарий обработки заказа

```
Создан
↓
Подтвержден
↓
Оплата получена
↓
В работе
↓
Отправлен
```

---

# Альтернативные сценарии

## Требуется уточнение

```
Создан
↓
Требует уточнения
↓
Подтвержден
```

## Отмена заказа

Отмена может произойти на любом этапе до отправки заказа:

```
Создан → Отменен
Подтвержден → Отменен
Оплата получена → Отменен
В работе → Отменен
```

---

# Действия пользователя (не являются статусами)

Следующие действия инициируют переходы между статусами:

- Подтвердить заказ  
- Запросить уточнение  
- Предоставить уточнение  
- Оплатить заказ  
- Отменить заказ
