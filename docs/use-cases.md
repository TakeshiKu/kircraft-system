# Use Cases Telegram-бота Kircraft

## 1. Назначение документа

Документ фиксирует пользовательские сценарии Telegram-бота Kircraft.

Цели документа:

- зафиксировать пользовательские сценарии системы
- описать поведение системы на уровне системного анализа
- сформировать единый перечень Use Cases
- использовать документ как основу для дальнейшей детализации системы

Документ основан на прототипе интерфейса Telegram-бота.

---

# 2. Область системы

В документе описывается пользовательское взаимодействие с Telegram-ботом Kircraft.

Система предоставляет возможность:

- просматривать каталог изделий
- просматривать изделия по коллекциям
- просматривать новинки
- просматривать карточки товаров
- настраивать параметры изделия
- добавлять товары в корзину
- управлять корзиной
- оформлять заказ
- получать уведомления о состоянии заказа
- просматривать галерею мастерской
- просматривать информацию о мастерской
- переходить к внешним каналам коммуникации

---

# 3. Граница системы

В Use Cases описывается поведение Telegram-бота как пользовательского интерфейса.

В документе не рассматриваются:

- архитектура системы
- база данных
- API
- админ-панель
- внутренняя логика обработки данных
- техническая реализация интеграций

---

# 7. Подробное описание Use Cases

## UC-01 Просмотр каталога изделий

| Раздел | Содержание |
|------|-----------|
| Код | UC-01 |
| Название | Просмотр каталога изделий |
| Описание | Клиент открывает каталог, выбирает категорию изделий и просматривает доступные товары выбранной категории для последующего перехода к карточке заинтересовавшего изделия |
| Акторы | Клиент, Система |
| Предусловия | Клиент открыл Telegram-бот Kircraft. Клиент находится в главном меню. В системе доступны категории каталога |
| Основной сценарий | 1. Клиент выбирает раздел «Каталог». |
|  | 2. Система отображает список категорий изделий. |
|  | 3. Клиент выбирает категорию изделий. |
|  | 4. Система отображает изделия выбранной категории. |
|  | 5. Клиент просматривает изделия выбранной категории. |
|  | 6. Система предоставляет возможность перейти к просмотру выбранного изделия. |
| Альтернативные сценарии | **3a. Клиент возвращается в главное меню** |
|  | 3a1. Клиент выбирает переход в главное меню. |
|  | 3a2. Система отображает главное меню. |
|  | 3a3. Сценарий завершается. |
|  |  |
|  | **3b. В выбранной категории отсутствуют товары** |
|  | 3b1. Система отображает сообщение об отсутствии товаров в выбранной категории. |
|  | 3b2. Сценарий завершается. |
|  |  |
|  | **5a. Клиент переходит к просмотру выбранного изделия** |
|  | 5a1. Клиент продолжает работу с выбранным изделием. |
|  | 5a2. Сценарий продолжается в UC-04 «Просмотр карточки изделия». |
| Исключения | Каталог недоступен для отображения. Система не может отобразить изделия выбранной категории |
| Постусловия | Система отобразила изделия выбранной категории. Клиент завершил просмотр каталога или перешел к просмотру выбранного изделия |
