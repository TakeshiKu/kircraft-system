# UC-01 Просмотр каталога изделий

## Актор

Клиент

## Описание

Клиент открывает каталог изделий в Telegram-боте и может просматривать доступные товары.

## Предусловия

- Клиент открыл Telegram-бот
- Каталог изделий загружен в систему

## Основной сценарий

1. Клиент открывает Telegram-бот.
2. Клиент выбирает пункт **Каталог**.
3. Система отображает список категорий изделий.
4. Клиент выбирает категорию.
5. Система отображает список изделий.

## Результат

Клиент видит доступные изделия.

---

# UC-02 Просмотр карточки изделия

## Актор

Клиент

## Описание

Клиент открывает карточку изделия для просмотра подробной информации.

## Предусловия

- Клиент находится в каталоге изделий

## Основной сценарий

1. Клиент выбирает изделие.
2. Система отображает карточку изделия.
3. Система показывает:

- название
- описание
- цену
- фотографии
- параметры изделия

## Результат

Клиент получает полную информацию об изделии.

---

# UC-03 Выбор параметров изделия

## Актор

Клиент

## Описание

Клиент выбирает параметры изделия перед добавлением в корзину.

## Предусловия

- Открыта карточка изделия

## Основной сценарий

1. Клиент выбирает параметры изделия.
2. Клиент выбирает количество.
3. Система пересчитывает стоимость.
4. Клиент нажимает **Добавить в корзину**.

## Результат

Товар добавлен в корзину.

---

# UC-04 Добавление изделия в корзину

## Актор

Клиент

## Описание

Клиент добавляет выбранное изделие в корзину.

## Основной сценарий

1. Клиент нажимает **Добавить в корзину**.
2. Система сохраняет товар в корзине клиента.
3. Система показывает уведомление о добавлении товара.

## Результат

Товар добавлен в корзину.

# UC-05 Просмотр корзины

## Актор

Клиент

## Описание

Клиент открывает корзину и просматривает выбранные товары.

## Основной сценарий

1. Клиент открывает раздел **Корзина**.
2. Система отображает список товаров.
3. Система показывает:

- название товара
- параметры

# UC-06 Оформление заказа

## Актор

Клиент

## Описание

Клиент оформляет заказ на основе товаров в корзине.

## Предусловия

- В корзине есть товары.

## Основной сценарий

1. Клиент нажимает **Оформить заказ**.
2. Система запрашивает контактные данные.
3. Клиент вводит:

- имя
- телефон
- город
- адрес доставки
- комментарий.

4. Клиент подтверждает заказ.
5. Система создает заказ.

## Результат

Заказ зарегистрирован в системе.

---

# UC-07 Подтверждение заказа

## Актор

Мастер

## Описание

Мастер получает новый заказ и принимает решение о его подтверждении.

## Основной сценарий

1. Система отправляет мастеру уведомление о новом заказе.
2. Мастер просматривает заказ.
3. Мастер выбирает действие:

- подтвердить заказ
- запросить уточнение
- отклонить заказ.

## Результат

Заказ получает новый статус.

---

# UC-08 Уточнение параметров заказа

## Актор

Мастер

## Описание

Мастер запрашивает дополнительную информацию у клиента.

## Основной сценарий

1. Мастер выбирает действие **Запросить уточнение**.
2. Мастер вводит комментарий.
3. Система отправляет сообщение клиенту.

## Результат

Клиент получает запрос на уточнение.

---

# UC-09 Оплата заказа

## Актор

Клиент

## Описание

Клиент оплачивает подтвержденный заказ.

## Основной сценарий

1. Клиент получает сообщение о подтверждении заказа.
2. Клиент выбирает оплату.
3. Клиент выполняет оплату.

## Результат

Система фиксирует оплату заказа.

---

# UC-10 Изменение статуса заказа

## Актор

Мастер

## Описание

Мастер обновляет статус заказа.

## Основной сценарий

Мастер может установить один из статусов:

- подтвержден
- требуется уточнение
- в работе
- отправлен
- завершен.

## Результат

Система обновляет статус заказа.

---

# UC-11 Уведомление клиента

## Актор

Система

## Описание

Система уведомляет клиента об изменении статуса заказа.

## Основной сценарий

1. Статус заказа изменяется.
2. Система отправляет сообщение клиенту.
3. Клиент получает уведомление.

## Результат

Клиент информирован о статусе заказа.
