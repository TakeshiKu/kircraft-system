# YooKassa Payment Integration

## 1. Общее описание

### Что это
Интеграция системы Kircraft с платежной системой YooKassa для приема онлайн-платежей.

### Для кого
Пользователи системы Kircraft (клиенты), оплачивающие заказ через клиентский интерфейс системы.

### Зачем
Реализовать возможность онлайн-оплаты заказа через YooKassa с автоматическим обновлением статуса платежа и заказа в системе.

### Краткий сценарий
1. Пользователь инициирует оплату заказа.
2. Backend системы создает платеж в YooKassa.
3. Пользователь перенаправляется на страницу оплаты YooKassa.
4. После завершения оплаты YooKassa отправляет webhook в систему.
5. Backend обновляет статус платежа и заказа.

## 2. Авторизация при работе с YooKassa API

API YooKassa использует серверную аутентификацию на основе механизма **HTTP Basic Auth**.

Каждый запрос к API должен содержать учетные данные магазина:

- `shopId` — идентификатор магазина  
- `secretKey` — секретный ключ API

Формат передачи авторизации:

Authorization: Basic base64(:)

Все обращения к API YooKassa выполняются только со стороны **Backend API Kircraft**.  
Клиентские интерфейсы (Telegram Bot, Web Interface, Telegram Mini App) не взаимодействуют с API YooKassa напрямую и не имеют доступа к учетным данным интеграции.

### Конфигурация Backend API

В конфигурации Backend API должны храниться следующие параметры:

- `YOOKASSA_SHOP_ID`
- `YOOKASSA_SECRET_KEY`
- `YOOKASSA_API_BASE_URL`

Пример базового URL:

https://api.yookassa.ru/v3

Хранение конфигурации определяется архитектурой Backend API и может быть реализовано через переменные окружения, конфигурационный файл или иное защищенное хранилище.

### Дополнительные требования

При выполнении операции создания платежа (`POST /payments`) должен передаваться заголовок:

Idempotence-Key

Заголовок используется для защиты от повторного создания одного и того же платежа при повторной отправке запроса.

# 3. Ограничения внешней системы

- взаимодействие только через HTTPS
- обязательная авторизация API (ShopID + Secret Key)
- webhook должен быть доступен публично
- идемпотентность запросов при создании платежа
- возможные ограничения по rate limit

# 4. Сценарий работы (Use Case)

## Пользователи

- Client (пользователь системы)

## Внутренние системы

- Client Interface (Telegram Bot / Web / Mini App)
- Backend API
- Database

## Внешние системы

- YooKassa API

---

## Входные данные

- order_id
- сумма заказа
- валюта
- описание заказа

---

## Предусловие

- заказ существует в системе
- статус заказа = `CREATED`
- заказ еще не оплачен

---

## Основной алгоритм работы

1. Пользователь нажимает кнопку **Оплатить заказ** в клиентском интерфейсе.
2. Client Interface отправляет запрос в Backend API на создание платежа.
3. Backend API проверяет существование и статус заказа.
4. Backend API создает запись в таблице `payments` со статусом `PENDING`.
5. Backend API отправляет запрос **Create Payment** в YooKassa API.
6. YooKassa возвращает:
   - `payment_id`
   - `confirmation_url`
7. Backend сохраняет `payment_id` в таблице `payments`.
8. Backend возвращает `confirmation_url` клиентскому интерфейсу.
9. Пользователь перенаправляется на страницу оплаты YooKassa.
10. Пользователь выполняет оплату.
11. YooKassa отправляет webhook уведомление о статусе платежа.
12. Backend принимает webhook.
13. Backend проверяет подпись webhook.
14. Backend обновляет статус платежа в таблице `payments`.
15. Backend обновляет статус заказа в таблице `orders`.

---

## Альтернативные сценарии и обработка ошибок

### Ошибка создания платежа

- YooKassa возвращает ошибку
- Backend фиксирует ошибку
- пользователю возвращается сообщение о невозможности создать платеж

### Пользователь не завершил оплату

- платеж остается в статусе `pending`
- заказ остается в статусе `created`

### Ошибка webhook

- webhook логируется
- система пытается повторно обработать событие

---

## Результат

В системе фиксируется:

- статус платежа
- идентификатор платежа YooKassa
- обновленный статус заказа

# 6. Используемые API-методы

## Создание платежа

Метод:

POST `/payments`

Назначение:

Создание платежа в системе YooKassa.

---

## Получение информации о платеже

Метод:

GET `/payments/{payment_id}`

Назначение:

Получение статуса платежа.

---

## Webhook уведомления

Endpoint системы:

POST `/api/payments/yookassa/webhook`

Назначение:

Получение уведомлений о статусе платежа.
