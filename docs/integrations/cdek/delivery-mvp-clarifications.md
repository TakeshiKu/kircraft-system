# Доставка CDEK (MVP): уточнения для backend

Назначение документа:
– документ фиксирует дополнительные правила реализации интеграции доставки CDEK для MVP
– не расширяет и не изменяет контракт docs/api/modules/delivery-api.md
– используется backend-разработкой как уточнение поведения системы

Документ дополняет контракт `docs/api/modules/delivery-api.md` и не заменяет его. При расхождении приоритет у `delivery-api.md`.

---

## 1. Источник данных

- Источником данных о ПВЗ и расчётах является **API CDEK** (вызовы выполняет только backend).
- **Кэширование** или **локальное хранение** на стороне backend допустимы как внутренняя оптимизация.
- Кэш и любое локальное хранение **не меняют** структуру и содержание ответов Delivery API для клиента (те же поля и семантика, что в контракте).

---

## 2. Параметры расчёта

- **Город получателя** — из тела запроса клиента (`POST /delivery/calculate`, поле `city`).
- **Параметры отправления** (вес, габариты и иные атрибуты, требуемые CDEK для тарифа) определяются **только backend** на основе **текущей активной корзины** (и при необходимости каталога); клиент их не передаёт в Delivery API.

---

## 3. Актуальность расчёта

- Каждый новый успешный или завершённый **`POST /delivery/calculate`** делает **предыдущий** результат расчёта **неактуальным** для целей выбора ПВЗ.
- **Selected Delivery** сбрасывается при **каждом** новом вызове `POST /delivery/calculate` (в соответствии с правилами сброса в `delivery-api.md`).
- Выбор ПВЗ (`POST /delivery/select`) возможен **только** из **последнего успешного** результата расчёта для текущего checkout.

---

## 4. Хранение выбранной доставки

- **Selected Delivery** хранится во **временном состоянии checkout**, привязанном к **текущей активной корзине** клиента.
- Состояние **перезаписывается** при каждом новом расчёте (`calculate`) или при успешном выборе доставки (`select`).

---

## 5. Валидация выбора ПВЗ

- Если `pickup_point_id` **не относится** к последнему успешному результату расчёта, система возвращает ошибку **`pickup_point_not_found`** (HTTP 404 по контракту модуля) и **не сохраняет** выбор.
