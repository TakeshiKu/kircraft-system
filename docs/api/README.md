# Kircraft Backend API

## 1. Назначение документа

Данный раздел описывает контракт backend API системы Kircraft.

API предназначен для:
- клиентских интерфейсов (Web, Telegram-бот, Mini App)
- административного интерфейса
- интеграционного взаимодействия с внешними системами

Документация фиксирует:
- структуру API
- правила взаимодействия
- контракты запросов и ответов
- единые соглашения и ограничения

Документ используется как:
- основа для разработки backend
- ориентир для frontend и интеграций
- источник истины при сопровождении системы

---

## 2. Область ответственности API

Backend API Kircraft отвечает за:

- управление каталогом изделий
- управление корзиной
- оформление и обработку заказов
- расчет и выбор доставки
- инициацию и обработку оплаты
- управление клиентами и их идентификацией
- формирование событий и уведомлений
- интеграцию с внешними сервисами (платежи, доставка, автоматизация)

API не включает:
- логику пользовательского интерфейса
- внутреннюю техническую реализацию backend
- детальное описание внешних API

---

## 3. Версионирование API

API версионируется через URL-префикс:

/api/v1

Принципы версионирования:
- изменения, нарушающие обратную совместимость, требуют новой версии (`v2`)
- изменения без нарушения контракта добавляются в текущую версию
- устаревшие (deprecated) методы могут временно поддерживаться в рамках версии

---

## 4. Общие принципы проектирования

API построен на следующих принципах:

- модульная структура (разделение по доменным областям)
- единые соглашения для всех методов
- явные границы ответственности модулей
- предсказуемое поведение методов
- единый формат ошибок
- идемпотентность для критичных операций (например, платежи)

Взаимодействие осуществляется через HTTP API с использованием JSON.

## 6. Модули API

### Cart API
Работа с корзиной клиента:
- получение корзины
- добавление товаров
- изменение количества
- удаление позиций

### Delivery API
Работа с доставкой:
- расчет стоимости
- получение списка ПВЗ
- выбор и сохранение доставки

### Order API
Управление заказами:
- создание заказа
- получение информации о заказе
- отмена заказа
- список заказов клиента

### Payment API
Работа с оплатой:
- создание платежа
- получение статуса оплаты
- обработка webhook платежной системы

### Customer API
Работа с клиентом:
- идентификация клиента
- управление контактными данными
- связь с внешними аккаунтами

### Admin Order API
Административные операции:
- управление статусами заказов
- уточнение данных заказа
- перевод заказа по жизненному циклу

## 7. Взаимодействие с внешними системами

Backend API интегрируется со следующими системами:
- платежные сервисы (YooKassa)
- сервисы доставки (CDEK)
- платформа автоматизации (n8n)
- мессенджеры (через Bot Gateway)

## 9. Принципы развития API

При развитии API соблюдаются следующие правила:

- новые функции добавляются в рамках существующих модулей
- расширение не нарушает существующие контракты
- изменения согласуются с моделью данных и архитектурой
- сохраняется модульная структура

Документация обновляется синхронно с изменениями API и остается актуальным источником описания системы.
