# Cart API

## 1. Назначение модуля

Cart API предназначен для работы с текущей корзиной клиента.

Модуль обеспечивает:
- получение текущего состояния корзины клиента
- добавление товара в корзину
- изменение количества позиции корзины
- удаление позиции корзины
- очистку корзины

Cart API используется клиентскими интерфейсами системы:
- Web
- Telegram-бот
- Mini App
- другие клиентские каналы

---

## 2. Границы ответственности модуля

Cart API отвечает за:
- управление текущей активной корзиной клиента
- управление составом корзины
- хранение выбранных параметров товара в составе позиции корзины
- предоставление клиенту актуального состояния корзины
- расчет итогов корзины на основе актуальных данных товаров

Cart API не отвечает за:
- управление каталогом товаров
- изменение данных товара
- расчет и выбор доставки
- оформление заказа
- создание и сопровождение оплаты
- административное управление корзинами

---

## 3. Связанные сущности

Модуль использует следующие сущности модели данных:
- `carts`
- `cart_items`
- `cart_item_selected_values`

Модуль зависит от следующих сущностей:
- `customers`
- `products`
- `product_photos`
- `product_parameters`
- `product_parameter_values`
- `product_available_parameters`
- `product_available_parameter_values`

Связь с заказами:
- корзина используется как источник данных при создании заказа
- после создания заказа активная корзина переходит в статус `converted`
- логика создания заказа не входит в область ответственности Cart API

---

## 4. Правила работы корзины

### 4.1 Общие правила

- Cart API работает только с активной корзиной текущего клиента
- клиент не передает `customer_id` или `cart_id` для выбора корзины
- backend определяет клиента из контекста запроса
- backend определяет и возвращает активную корзину клиента

### 4.2 Активная корзина

- в клиентском контексте используется одна активная корзина
- если активная корзина отсутствует, backend создает ее автоматически
- пустая корзина не считается ошибкой
- методы Cart API работают только с корзиной в статусе `active`

### 4.3 Правила позиций корзины

- каждая позиция корзины содержит:
  - товар
  - количество
  - выбранные параметры товара (массив, может быть пустым)

- если клиент добавляет тот же товар с тем же набором выбранных параметров:
  - backend увеличивает количество существующей позиции корзины

- если клиент добавляет тот же товар с другим набором выбранных параметров:
  - backend создает отдельную позицию корзины

При сравнении набора выбранных параметров:
- учитываются пары `parameter_id` + `parameter_value_id`
- порядок элементов в массиве не учитывается
- дублирование `parameter_id` в одном наборе не допускается

### 4.4 Правила по товарам и параметрам

- в корзину добавляются только товары, доступные для продажи в клиентском контексте
- набор выбранных параметров должен быть допустим для конкретного товара
- обязательные параметры товара должны быть переданы полностью, если такие параметры заданы в модели данных

backend проверяет:
- существование товара
- доступность товара
- допустимость каждого параметра для товара
- допустимость каждого выбранного значения параметра для товара
- полноту обязательного набора параметров

### 4.5 Правила по цене

- Cart API не фиксирует snapshot цены в составе позиции корзины
- цена позиции рассчитывается на основе актуальных данных товара на момент запроса
- итоговая фиксация цены выполняется на этапе оформления заказа и относится к Order API

### 4.6 Недоступные товары в существующей корзине

Если товар, ранее добавленный в корзину, стал недоступен:
- позиция сохраняется в корзине
- позиция помечается как недоступная для оформления
- `unit_price` и `line_total` возвращаются всегда и используются при расчете `totals`, даже если `availability.is_available = false`
- изменение количества такой позиции не допускается
- удаление позиции остается доступным
- повторное добавление такого товара в корзину запрещено

Поведение оформления заказа с недоступными позициями определяется в Order API.

## 5. Endpoint overview

Модуль включает следующие методы:
- `GET /cart` - получить текущее состояние корзины клиента
- `POST /cart/items` - добавить товар в корзину
- `PATCH /cart/items/{cart_item_id}` - изменить количество позиции корзины
- `DELETE /cart/items/{cart_item_id}` - удалить позицию корзины
- `DELETE /cart/items` - очистить корзину

## 6. Контракты методов

### 6.1 Получение текущей корзины

#### Метод и путь

`GET /cart`

#### Назначение

Возвращает текущее состояние активной корзины клиента.

Если активная корзина отсутствует, backend создает пустую активную корзину.

#### Request

Body отсутствует.

#### Response 200

```json
{
  "data": {
    "cart_id": "cart_001",
    "status": "active",
    "items": [
      {
        "cart_item_id": "cart_item_001",
        "product": {
          "product_id": "product_001",
          "product_name": "Обложка для паспорта",
          "description_short": "Кожаная обложка ручной работы",
          "main_photo_path": "/media/products/passport-cover-main.jpg",
          "status": "active"
        },
        "availability": {
          "is_available": true,
          "reason": null
        },
        "quantity": 2,
        "unit_price": 150000,
        "line_total": 300000,
        "selected_values": [
          {
            "parameter_id": "parameter_001",
            "parameter_name": "Цвет",
            "parameter_value_id": "value_005",
            "value_name": "Черный"
          }
        ],
        "created_at": "2026-03-19T18:00:00Z"
      }
    ],
    "totals": {
      "items_count": 2,
      "positions_count": 1,
      "items_total": 300000,
      "currency": "RUB"
    },
    "created_at": "2026-03-19T17:55:00Z",
    "updated_at": "2026-03-19T18:00:00Z"
  },
  "meta": {
    "request_id": "uuid"
  }
}
```

#### Правила ответа

- `items` содержит актуальный состав корзины
- `selected_values` всегда передается массивом и может быть пустым
- `unit_price` и `line_total` передаются в минимальных денежных единицах и всегда возвращаются, даже если `availability.is_available = false`
- пустая корзина возвращается с пустым массивом `items`
- `availability.is_available = false` означает, что позиция недоступна для оформления

#### Пример пустой корзины

```json
{
  "data": {
    "cart_id": "cart_001",
    "status": "active",
    "items": [],
    "totals": {
      "items_count": 0,
      "positions_count": 0,
      "items_total": 0,
      "currency": "RUB"
    },
    "created_at": "2026-03-19T17:55:00Z",

  },
  "meta": {
    "request_id": "uuid"
  }
}
```

### 6.2 Добавление товара в корзину

#### Метод и путь

`POST /cart/items`

#### Назначение

Добавляет товар в активную корзину клиента.

Если активная корзина отсутствует, backend создает ее автоматически.

Если в корзине уже есть позиция с тем же товаром и тем же набором параметров:
- backend увеличивает количество существующей позиции

#### Request

```json
{
  "product_id": "product_001",
  "quantity": 2,
  "selected_values": [
    {
      "parameter_id": "parameter_001",
      "parameter_value_id": "value_005"
    }
  ]
}
```

#### Поля request

- `product_id` - обязательное поле, идентификатор товара
- `quantity` - обязательное поле, целое число больше 0
- `selected_values` - необязательное поле, список выбранных параметров товара

#### Правила request

- `quantity` должно быть больше 0
- `selected_values` может отсутствовать или быть пустым массивом
- один и тот же `parameter_id` не должен повторяться в рамках одного запроса
- каждый `parameter_value_id` должен соответствовать переданному `parameter_id`
- backend проверяет допустимость набора параметров для товара

#### Response 200

```json
{
  "data": {
    "cart_item_id": "cart_item_001",
    "product": {
      "product_id": "product_001",
      "product_name": "Обложка для паспорта",
      "description_short": "Кожаная обложка ручной работы",
      "main_photo_path": "/media/products/passport-cover-main.jpg",
      "status": "active"
    },
    "availability": {
      "is_available": true,
      "reason": null
    },
    "quantity": 2,
    "unit_price": 150000,
    "line_total": 300000,
    "selected_values": [
      {
        "parameter_id": "parameter_001",
        "parameter_name": "Цвет",
        "parameter_value_id": "value_005",
        "value_name": "Черный"
      }
    ],
    "created_at": "2026-03-19T18:05:00Z",
    "totals": {
      "items_count": 2,
      "positions_count": 1,
      "items_total": 300000,
      "currency": "RUB"
    }
  },
  "meta": {
    "request_id": "uuid"
  }
}
```

#### Правила ответа

- response содержит актуальное состояние добавленной или обновленной позиции
- response содержит актуальные totals корзины
- `unit_price` и `line_total` возвращаются всегда, даже если `availability.is_available = false`
- операция не гарантирует создание нового ресурса: backend может обновлять существующую позицию (например, увеличивать количество)
- `selected_values` всегда передается массивом и может быть пустым

### 6.3 Изменение количества позиции корзины

#### Метод и путь

`PATCH /cart/items/{cart_item_id}`

#### Назначение

Изменяет количество существующей позиции корзины.

#### Request

```json
{
  "quantity": 3
}
```

#### Поля request

- `quantity` - обязательное поле, целое число больше 0

#### Правила request

- `quantity` должно быть больше 0
- если позиция не принадлежит текущему клиенту или позиция не существует, backend возвращает ошибку `404 cart_item_not_found`
- если позиция существует, но связанная корзина не в статусе `active`, backend возвращает ошибку `409 cart_not_active`
- если `availability.is_available = false`, изменение количества запрещено и возвращается ошибка `422 validation_error`
- в `error.details` указывается причина недоступности в поле `availability_reason` (значение из допустимых значений `availability.reason`)
- изменение других полей позиции через этот метод не допускается

#### Response 200

```json
{
  "data": {
    "cart_item_id": "cart_item_001",
    "product": {
      "product_id": "product_001",
      "product_name": "Обложка для паспорта",
      "description_short": "Кожаная обложка ручной работы",
      "main_photo_path": "/media/products/passport-cover-main.jpg",
      "status": "active"
    },
    "availability": {
      "is_available": true,
      "reason": null
    },
    "quantity": 3,
    "unit_price": 150000,
    "line_total": 450000,
    "selected_values": [
      {
        "parameter_id": "parameter_001",
        "parameter_name": "Цвет",
        "parameter_value_id": "value_005",
        "value_name": "Черный"
      }
    ],
    "created_at": "2026-03-19T18:05:00Z",
    "totals": {
      "items_count": 3,
      "positions_count": 1,
      "items_total": 450000,
      "currency": "RUB"
    }
  },
  "meta": {
    "request_id": "uuid"
  }
}
```

#### Правила ответа

- response содержит актуальные totals корзины
- `unit_price` и `line_total` возвращаются всегда, даже если `availability.is_available = false`
- `selected_values` всегда передается массивом и может быть пустым

#### Допущение

Удаление позиции через `quantity = 0` не поддерживается. Для удаления используется отдельный DELETE-метод.

---

### 6.4 Удаление позиции корзины

#### Метод и путь

`DELETE /cart/items/{cart_item_id}`

#### Назначение

Удаляет одну позицию из активной корзины клиента.

#### Request

Body отсутствует.

#### Response 204

Тело ответа отсутствует.

#### Правила

- переданный `cart_item_id` должен относиться к позиции текущего клиента
- удаление не затрагивает другие позиции корзины
- если позиция не принадлежит текущему клиенту или позиция не существует, backend возвращает ошибку `404 cart_item_not_found`
- если позиция существует, но связанная корзина не в статусе `active`, backend возвращает `409 cart_not_active`

---

### 6.5 Очистка корзины

#### Метод и путь

`DELETE /cart/items`

#### Назначение

Удаляет все позиции из активной корзины клиента.

#### Request

Body отсутствует.

#### Response 204

Тело ответа отсутствует.

#### Правила

- очищается только активная корзина текущего клиента
- сама корзина как сущность сохраняется
- после очистки корзина остается активной и пустой


- `product_archived`
- `product_inactive`
- `product_unavailable`

- используется для машинной интерпретации причины недоступности
- не предназначено для прямого отображения пользователю без слоя адаптации на клиенте

## 8. Ошибки модуля

Ошибки модуля соответствуют общему формату, описанному в `errors.md`.

### Возможные ошибки

#### 400 invalid_request

Некорректная структура запроса:
- отсутствует обязательное поле
- неверный тип данных
- дублируется параметр в `selected_values`

#### 401 unauthorized

Клиент не аутентифицирован.

#### 404 product_not_found

Товар с указанным идентификатором не существует.

#### 404 cart_item_not_found

Позиция корзины не найдена или не принадлежит текущему клиенту.

#### 409 cart_not_active

Позиция корзины принадлежит текущему клиенту, но связанная корзина не в статусе `active`.

#### 422 validation_error

Ошибка бизнес-валидации:
- товар существует, но недоступен для добавления в корзину
- `quantity` меньше 1
- позиция недоступна для изменения количества (`availability.is_available = false`) (причина: `error.details.availability_reason`)
- параметр недопустим для товара
- значение параметра недопустимо для товара
- обязательные параметры товара переданы не полностью

#### 500 internal_error

Внутренняя ошибка backend.

### Правила интерпретации ошибок

- если позиция не принадлежит текущему клиенту или позиция не существует, backend возвращает `404 cart_item_not_found`
- если позиция существует, но связанная корзина не в статусе `active`, backend возвращает `409 cart_not_active`
- отдельная ошибка `403 access_denied` для операций с позициями корзины не используется, чтобы не раскрывать информацию о существовании чужих ресурсов
- ошибки валидации состава товара и параметров возвращаются как `422 validation_error`

## 9. Правила безопасности

Для методов Cart API применяются правила, описанные в `security.md`.

Дополнительно:
- все методы Cart API работают только в контексте аутентифицированного клиента
- клиент работает только со своей активной корзиной
- backend проверяет принадлежность `cart_item_id` текущему клиенту
- backend не принимает от клиента цену, сумму позиции или итоговую сумму корзины
- доступность товаров и параметров определяется на стороне backend
- backend не доверяет составу выбранных параметров без проверки их допустимости для конкретного товара
- операции изменения корзины выполняются только для корзины в статусе `active`
- после каждой операции изменения корзины backend самостоятельно пересчитывает итоговые денежные значения корзины
