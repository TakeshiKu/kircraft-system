## UC-A5 Генерация черновика описания товара через AI

| Раздел | Содержание |
|------|-----------|
| Код | UC-A5 |
| Название | Генерация черновика описания товара через AI |
| Описание | Мастер передаёт параметры нового изделия, система формирует AI-черновик описания, мастер проверяет, при необходимости правит и подтверждает публикацию |
| Акторы | Мастер, Система, AI Assistant Module |
| Предусловия | Мастер аутентифицирован в бэкофисе. Мастер находится в режиме добавления или редактирования карточки товара. AI Assistant Module доступен |
| Основной сценарий | 1. Мастер вводит параметры изделия: тип, материал, размер, особенности. |
|  | 2. Мастер выбирает действие «Сгенерировать описание через AI». |
|  | 3. Система передаёт параметры товара в AI Assistant Module. |
|  | 4. AI Assistant Module формирует промпт и вызывает LLM Provider. |
|  | 5. AI Assistant Module логирует вызов: промпт, модель, версия, timestamp, результат, статус. |
|  | 6. Система отображает мастеру черновик описания, помеченный как AI-сгенерированный. |
|  | 7. Мастер просматривает и при необходимости редактирует текст. |
|  | 8. Мастер подтверждает публикацию описания. |
|  | 9. Система фиксирует подтверждение мастера и публикует описание в карточке товара. |
| Альтернативные сценарии | **2a. Мастер запрашивает повторную генерацию** |
|  | 2a1. Мастер выбирает «Сгенерировать заново». |
|  | 2a2. Сценарий возвращается к шагу 3. |
|  |  |
|  | **6a. Мастер отказывается от черновика** |
|  | 6a1. Мастер закрывает черновик без подтверждения. |
|  | 6a2. Система не публикует описание. |
|  | 6a3. Мастер продолжает заполнение карточки товара вручную. |
|  |  |
|  | **8a. Мастер существенно редактирует черновик перед подтверждением** |
|  | 8a1. Описание сохраняется как итоговое содержимое, подтверждённое мастером. |
|  | 8a2. Маркировка «AI-сгенерированный» снимается, поскольку результат одобрен мастером после правок. |
| Исключения | LLM Provider недоступен или вернул ошибку. |
|  | В этом случае система отображает мастеру невозможность AI-операции и предлагает заполнить описание вручную; работа карточки товара не блокируется. |
| Постусловия | В случае подтверждения — описание товара опубликовано в карточке. В случае отказа — описание не сохранено, мастер продолжает работу в обычном режиме. Вызов LLM зафиксирован в логе AI-вызовов |

Связанные BR: BR-AI-01, BR-AI-03, BR-AI-04, BR-AI-06.
