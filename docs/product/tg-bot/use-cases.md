# Use Cases Telegram-бота Kircraft

## 1. Назначение документа

Документ фиксирует пользовательские сценарии Telegram-бота Kircraft.

Цели документа:

- зафиксировать пользовательские сценарии системы
- описать поведение системы на уровне системного анализа
- сформировать единый перечень Use Cases
- использовать документ как основу для дальнейшей детализации системы

Документ основан на прототипе интерфейса Telegram-бота.

---

# 2. Область системы

В документе описывается пользовательское взаимодействие с Telegram-ботом Kircraft.

Система предоставляет возможность:

- просматривать каталог изделий
- просматривать изделия по коллекциям
- просматривать новинки
- просматривать карточки товаров
- настраивать параметры изделия
- добавлять товары в корзину
- управлять корзиной
- оформлять заказ
- получать уведомления о состоянии заказа
- просматривать галерею магазина
- просматривать информацию о магазине
- переходить в Telegram-чат магазина
- переходить в Telegram-канал магазина

---

# 3. Граница системы

В Use Cases описывается поведение Telegram-бота как пользовательского интерфейса.

В документе не рассматриваются:

- архитектура системы
- база данных
- API
- админ-панель
- внутренняя логика обработки данных
- техническая реализация интеграций

---

|  |  |

|  | 3a1. Система сохраняет статус заказа без изменений |
| Исключения | Не удалось загрузить данные заказа. Не удалось сохранить решение мастера |

## Общая схема пользовательского потока

Каталог / Коллекции / Новинки
→ Карточка изделия

→ Добавление в корзину
→ Управление корзиной
→ Оформление заказа

`awaiting_payment`  
→ `paid`  
→ `in_progress`

- `needs_clarification`
- `cancelled`
- `rejected`

# 8. Общие системные исключения

Следующие исключительные ситуации применимы ко всем пользовательским сценариям, если иное не указано в конкретном Use Case:

- временная недоступность сервиса
- ошибка загрузки данных
- ошибка сохранения данных
- потеря соединения во время выполнения сценария
- повторное выполнение действия после сбоя

При возникновении указанных ситуаций система должна:
- сообщить пользователю о невозможности завершить действие
- сохранить ранее введенные данные, если это возможно
- предложить повторить попытку
